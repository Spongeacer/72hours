# 72Hours 设计理念

> **涌现式叙事游戏的核心设计哲学**
> 
> 版本: 2.0.0 | 最后更新: 2025-02-28

---

## 🎯 核心设计原则

### 1. 涌现式叙事 (Emergent Narrative)

**不是预设剧情，而是故事自己长出来。**

传统叙事游戏：
```
作者写剧本 → 玩家选择分支 → 到达预设结局
```

72Hours的涌现式叙事：
```
物理场 + 在场者 + 环境信号 + 记忆 → 故事自然流淌
```

每一个NPC都有自己的执念、记忆和行为逻辑，在物理引擎的驱动下自然交互，产生不可预测的叙事涌现。

#### 涌现的层次

| 层次 | 涌现来源 | 示例 |
|------|----------|------|
| 微观 | NPC个体行为 | 恐惧的村民选择逃跑或攻击 |
| 中观 | NPC间互动 | 两个有恩怨的NPC相遇产生冲突 |
| 宏观 | 集体情绪 | 恐慌蔓延导致群体行为 |
| 叙事 | AI共振 | 基于情境生成的独特叙事文本 |

### 2. 玩家作为催化剂 (Player as Catalyst)

**你不是故事的主角，而是催化剂——在场，影响，但故事自己流淌。**

```
传统RPG: 玩家 = 主角 → 故事围绕玩家展开
72Hours: 玩家 = 催化剂 → 玩家改变"场"，故事因你而变
```

#### 催化剂效应

- **在场即影响**：你的存在本身就改变了周围的物理场
- **选择有后果**：但不是决定论式的后果，而是涌现式的涟漪
- **不可控**：你无法控制NPC的行为，只能影响概率

### 3. 引力模型物理引擎

基于 `F = G × M₁ × M₂ / r² × (1 + Ω × P)` 的引力公式，驱动NPC移动、互动和叙事焦点选择。

#### 质量 (Mass)

```
M = B(基础) + S(叙事) + K(关系) + O(道具)
```

| 质量类型 | 说明 | 示例 |
|----------|------|------|
| 基础质量 (B) | 身份决定的基础值 | 洪秀全=10，普通村民=3 |
| 叙事质量 (S) | 故事重要性 | 关键剧情NPC质量更高 |
| 关系质量 (K) | K值总和 | 关系网越大质量越大 |
| 道具质量 (O) | 携带物品 | 神兵利器增加质量 |

#### 压强 (Pressure)

压强代表环境的紧张程度，1-20范围：

| 压强范围 | 状态 | 效果 |
|----------|------|------|
| 1-5 | 平静 | 正常互动 |
| 6-10 | 紧张 | 互动频率增加 |
| 11-15 | 高压 | 暴力行为概率上升 |
| 16-20 | 临界 | 失范状态，社会规范瓦解 |

#### Ω (Omega) - 历史必然感

Ω代表历史事件的必然性，随时间自动增长：

```
Ω < 12: 线性增长（每回合+0.4）
Ω ≥ 12: 指数增长（每回合×1.02）
```

Ω达到阈值时触发历史锚点事件。

### 4. AI驱动的选择生成

使用AI API基于玩家特质、执念和当前情境，生成个性化的反应选择。

#### 选择类型

| 类型 | 驱动 | 示例 |
|------|------|------|
| obsession | 执念 | "你想起自己的执念，这让你做出了选择" |
| trait | 特质 | "你的'勇敢'特质让你选择了这样的方式" |
| instinct | 本能 | "你凭着本能做出了反应" |
| context | 情境 | "基于当前环境，你决定..." |

#### 选择生成流程

```
玩家状态 + NPC行为 + 环境场 → AI Prompt → 3个独特选择
```

每个玩家的选择都是独特的，由AI实时涌现生成。

## 🎭 社会心理学基础

游戏的行为模型基于以下社会心理学理论：

### 1. 社会交换理论 (Social Exchange Theory)

**资源稀缺下的成本-收益计算**

```
贪婪特质 + 恐惧状态 + 高压环境 → 抢夺行为概率上升
```

### 2. 挫折-攻击假说 (Frustration-Aggression Hypothesis)

**目标受阻引发攻击**

```
高攻击性 + 目标受阻 → 暴力行为
```

### 3. 依恋理论 (Attachment Theory)

**安全基地寻求**

```
高K值 + 恐惧状态 → 寻求亲近
```

### 4. 互惠规范 (Norm of Reciprocity)

**社会交换的道德基础**

```
慈悲特质 + 深度关系 → 无条件给予
```

### 5. 失范理论 (Anomie Theory) - Durkheim

**高压下的规范瓦解**

```
恐惧 > 14 + 饥饿 > 12 → 失范状态
```

在失范状态下，NPC可能做出平时不会做的行为。

### 6. 信息缺口理论 (Information Gap Theory)

**好奇驱动探索**

```
好奇特质 + 安全环境 → 偷听/探索行为
```

### 7. 社会认同理论 (Social Identity Theory)

**群体身份影响行为**

```
勇敢特质 + 高压环境 → 保护行为
```

## 🏛️ 历史锚点系统

5个关键历史事件由Ω（历史必然感）驱动触发：

### 事件链

```
暗流涌动 → 官兵搜查 → 天父下凡 → 万寿祝寿 → 金田起义
   Ω=2       Ω≥6        Ω≥12       Ω≥16       Ω≥20/36回合
```

### 锚点特性

- **必然性**：历史事件一定会发生（Ω驱动）
- **不可预测性**：具体形式和玩家体验是涌现的
- **玩家影响**：玩家的行为可能加速或减缓事件进程

### 蝴蝶效应

玩家的每个选择都可能产生微小但累积的影响：

```
选择 → 状态变化 → 物理场变化 → NPC行为变化 → 叙事变化
```

虽然单个选择的影响很小，但累积起来可能改变故事的走向。

## 🎮 游戏循环

### 核心循环

```
┌─────────────────────────────────────────────────────┐
│  1. 物理场更新 (压强↑, Ω↑)                          │
│     ↓                                               │
│  2. NPC移动 (引力驱动)                              │
│     ↓                                               │
│  3. 聚光灯选择 (最大引力NPC)                        │
│     ↓                                               │
│  4. 行为涌现 (社会心理学模型)                       │
│     ↓                                               │
│  5. 叙事生成 (AI/离线共振)                          │
│     ↓                                               │
│  6. 选择呈现 (基于物理距离和K值)                    │
│     ↓                                               │
│  7. 玩家选择 → 回到1                               │
└─────────────────────────────────────────────────────┘
```

### 回合结构

每个回合代表2小时的游戏时间，共36回合（72小时）。

| 阶段 | 活动 |
|------|------|
| 物理更新 | 压强、Ω增长；NPC移动 |
| 叙事生成 | AI生成或离线共振文本 |
| 选择呈现 | 基于距离和K值的选择 |
| 玩家决策 | 选择并影响下一回合 |

## 🌊 叙事美学

### 共振式写作

不是描述"发生了什么"，而是营造"感觉是什么"。

#### 示例对比

**传统描述**：
```
压强很高，大家都很害怕。洪秀全正在策划起义。
```

**共振式描述**：
```
> 空气像凝固的血。
> 远处有火光，不是灯笼的颜色。
> 有人在低语，但你听不清内容。
> 你感到一种无法逃避的命运感。
```

### 暗示而非说明

- ❌ "他很贪婪，所以想要你的东西"
- ✅ "他的目光在你的包袱上停留了三秒"

### 第二人称沉浸

所有叙事都采用第二人称，增强代入感：

```
> 你感到后背发凉。
> 你不知道历史已经开始了。
```

## 🔧 设计决策记录

### 为什么用引力模型？

1. **直观**：物理概念易于理解
2. **涌现性**：简单规则产生复杂行为
3. **可预测与随机的平衡**：引力可计算，但20%随机扰动保持不确定性

### 为什么用1-20范围？

1. **D20熟悉度**：TRPG玩家熟悉20面骰
2. **足够粒度**：20个等级足够区分状态
3. **计算简单**：避免浮点数精度问题

### 为什么36回合？

1. **历史准确**：1851年1月8日到11日约72小时
2. **游戏时长**：每回合2-3分钟，总时长适中
3. **叙事节奏**：36个节点足够讲述完整故事

### 为什么AI生成选择？

1. **独特性**：每个玩家的体验都不同
2. **涌现性**：AI可以创造设计者想不到的选择
3. **可扩展性**：易于添加新情境
