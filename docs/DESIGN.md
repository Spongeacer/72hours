# 72Hours 叙事引擎设计文档（v1.1 涌现版）

> 基于引力模型的涌现式历史叙事系统
> 核心机制：叙事对撞机 (Narrative Collider)
> 首个 Demo：太平天国金田起义（1851年1月8日-11日，72回合）

---

## 1. 核心设计理念

### 1.1 玩家作为催化剂

玩家不是故事的主角，而是**催化剂**——在场，影响，但故事自己流淌。

```
无玩家时：故事默默进行，无人知晓
有玩家时：玩家的存在改变周围的"场"，故事因你而变

玩家可能：
- 不说话，只是也在（陪伴催化）
- 给食物（恩情催化）
- 讲述恐惧（共鸣催化）
- 沉默磨刀（紧张催化）

故事因你而变，但不是由你决定。
```

### 1.2 涌现式叙事

**不是预设公式，而是故事自己长出来。**

```
输入：此刻的所有在场者 + 环境 + 信号 + 记忆
↓
共鸣（不是计算，是共振）
↓
故事自然流淌

示例：
> 雨水顺着他未收鞘的刀尖滴落。
> 你想起父亲。
> 他也想起父亲。
> 两人都没有说话。
```

**涌现的源头**：
- NPC的执念（此刻什么对他最重要）
- NPC的记忆（你让他想起谁）
- 环境信号（雨、血、远处的喊杀）
- 玩家的"气场"（你散发出什么）
- 纯粹的偶然（随机扰动，无因之果）

## 3. 物理引擎

### 3.1 引力公式

```
F = G × M₁ × M₂ / r² × P × Ω
```

| 符号 | 含义 | 说明 |
|------|------|------|
| G | 引力常数 | 待定 |
| M₁, M₂ | 双方质量 | M = B + S + K + O |
| r | 欧几里得距离 | 坐标差计算 |
| P | 环境压强 | P = P_history × P_traits |
| Ω | 全局因子 | 随倒计时指数增长 |

### 3.2 质量模型

```
M = B + S + K + O
```

| 分量 | 全称 | 获取方式 | 叙事含义 |
|------|------|----------|----------|
| B | Base（基础质量） | 角色预设 | 先天位格 |
| S | Story（叙事权重） | 经历事件累积 | 活得越久质量越高 |
| K | Knot（关系系数） | 交互次数 | 纠缠越深质量越大 |
| O | Object（道具加成） | 持有道具 | 关键道具瞬间暴增 |

### 3.3 环境压强

```
P = P_history × P_traits
```

**P_history（历史压强）**：随倒计时不可逆增长
**P_traits（特质扰动）**：激化型/缓冲型/连锁型特质影响

### 3.4 物理叙事现象

| 现象 | 机制 | 叙事效果 |
|------|------|----------|
| **引力捕获** | 路人S/K飙升 | 路人变生死之交 |
| **质量坍缩** | NPC死亡→M变负 | 心理创伤，无法回到相关坐标 |
| **双星系统** | 两高M NPC互吸 | 玩家被甩出，旁观对决 |
| **全局坍缩** | Ω指数增长 | 所有人撞向原点，命运交织 |

## 4. 行为模式库

**核心原则**：行为从库中自然涌现，具体叙事由当下共振生成。

### 4.1 六类行为

| 行为 | 核心驱动力 | 可能涌现的场景 |
|------|------------|----------------|
| 抢夺 | [贪婪]/恐慌/生存 | "他的手伸向你的腰间，眼神却在躲闪" |
| 冲突 | [嗜血]/仇恨/恐惧 | "刀光闪过，你感到胸口一凉" |
| 偷听 | [好奇]/[狡诈]/执念 | "他躲在门后，呼吸声比雨声还轻" |
| 聊天 | 孤独/寻求理解/特质共鸣 | "两个读书人，在乱世的油灯下谈起诗" |
| 请求 | 绝望/信任/执念驱动 | "他跪下，不是因为软弱，是因为别无选择" |
| 给予 | [慈悲]/报恩/执念 | "她把最后一块干粮塞给你，像母亲一样" |

### 4.2 行为涌现示例

**场景**：雨夜，破庙，逃兵李三

```
李三的状态：
- fear: 80（极度恐惧）
- hunger: 70（饥饿）
- 记忆：被玩家救助过（K=3）
- 执念：[苟活下去]

玩家的在场：
- 沉默地坐着（陪伴催化）
- 身上有血（紧张信号）
- 也发抖（共鸣催化）

可能涌现：
- 他扑上来抢食物（恐慌驱动的抢夺）
- 他哭泣，讲述家乡（恐惧驱动的聊天）
- 他沉默地靠过来（孤独驱动的陪伴）
- 他把刀递给你（绝望驱动的请求）

没有公式决定哪个，只有此刻的"感觉"对"感觉"的碰撞。
```

## 5. 历史锚点与事件系统

### 5.1 核心机制：事件+聚光灯+玩家 共同演绎

**不是分开处理，而是整体涌现。**

```
传统设计：事件 → NPC反应 → 玩家选择
本设计：事件 + 聚光灯NPC + 玩家状态 → AI共同演绎

示例：
- 事件：[官兵搜查]
- 聚光灯NPC：[清军营官]，特质[贪婪][嗜血]
- 玩家状态：[教徒]，怀里藏着[十字架]

AI共同演绎：
> 门被撞开。三个官兵站在门口，雨水顺着刀尖滴落。
> 领头的营官看着你，眼神里有贪婪，也有疲惫。
> "搜。"他说。
> 
> 你怀里藏着十字架。他的目光扫过你的胸口，停住了。
> 你知道他看见了。他也知道你知道。
> 
> 空气凝固。

不是"事件触发→NPC出现→玩家应对"的三段式，
而是三者共振，一次生成完整的叙事瞬间。
```

### 5.2 5个历史锚点

| 锚点 | 回合 | 事件 | 性质 | 动态条件 |
|------|------|------|------|----------|
| 1 | 1 | 暗流涌动 | 固定 | 开场，玩家醒来 |
| 2 | 24±6 | 官兵搜查 | 动态 | P>50提前；P<30延后 |
| 3 | 48±4 | 天父下凡 | 动态 | 会众质量总和>阈值提前 |
| 4 | 71 | 万寿祝寿 | 固定前奏 | 所有精英NPC强制聚集 |
| 5 | 72 | 金田起义 | 固定结局 | 历史大势，不可改变 |

### 5.3 非关键事件

随机触发，丰富世界，不影响历史大势：
- 外国传教士进村
- 官府征粮
- 瘟疫蔓延
- 流民涌入
- 乡绅宴会
- 天象异变

### 5.4 动态触发机制

**锚点2：官兵搜查**
- 提前：P > 50（局势紧张）或 玩家戾气值 > 70（暴力信号广播）
- 延后：P < 30（表面平静）且 玩家质量 M < 4（无威胁）

**锚点3：天父下凡**
- 提前：会众NPC质量总和 > 阈值（集体觉醒）或 杨秀清对玩家引力 F > 8
- 延后：会众分散（质量分散）或 杨秀清质量 M < 6（领袖失能）

### 5.3 锚点的涌现式呈现

**不是触发剧情，而是物理状态达到临界后的自然现象。**

```
锚点2涌现示例：

P=65，深夜，雨
玩家坐标附近突然刷新3个[官兵]NPC
他们的F值瞬间飙升（P调制）
聚光灯锁定其中一个

可能涌现：
- 他推门而入，刀尖滴雨（冲突）
- 他躲在门外偷听（偷听）
- 他敲门请求避雨（请求）
- 他径直走过，仿佛没看见（偶然）

"官兵搜查"不是预设剧情，是P和F达到阈值后，
[官兵]NPC自然聚集并产生高引力的涌现现象。
```

## 6. 提前结束（涌现式）

### 6.1 没有预设结局

**死亡不是"游戏结束"，而是故事的一种形态完成。**

```
示例：
> 刀光闪过，你感到胸口一凉。
> 倒下去的时候，你听到远处有人在喊："起义了！太平天国！"
> 你没有看到那一天。
>
> 【故事在此完成】
> 你的死亡成为金田起义的一个注脚。
> 72小时后，这里将血流成河。
```

### 6.2 涌现式提前结束

| 状态组合 | 可能涌现 |
|----------|----------|
| injury=致命 + 无救治 | 死亡 |
| fear=100 + 连续逃跑成功 | 逃离金田（永久离开坐标系） |
| pressure=100 + 精神崩溃 | 疯癫（故事继续但玩家失控） |
| 被囚禁 + 无法逃脱 | 困于狱中（空间锁定） |

### 6.3 死亡后的信息

- 显示如果活到第72回合可能看到的景象
- 显示你的死亡如何影响其他NPC（质量坍缩、执念改变）
- 显示历史如何继续（起义是否爆发、谁记住了你）

## 9. 设计原则总结

1. **玩家是催化剂**：在场即影响，但不控制
2. **故事自己涌现**：有框架，无预设
3. **物理驱动叙事**：F/P/M 决定可能性，不决定结果
4. **历史大势不可变**：洪秀全会出现，起义会爆发
5. **个人命运可塑造**：谁与你同行，由你的存在决定
6. **死亡是完成**：不是失败，是故事的一种形态
