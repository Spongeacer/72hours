# 设计理念 vs 代码实现对比分析

## 1. 核心设计理念 - 实现状态

### 1.1 玩家作为催化剂
- **设计理念**: 玩家不是主角，而是催化剂——在场、影响，但故事自己流淌
- **代码实现**: ❌ **写死逻辑** - 玩家是中心，所有NPC围绕玩家转动
- **问题**: 没有实现"故事默默进行，无人知晓"的独立NPC行为

### 1.2 涌现式叙事
- **设计理念**: 不是预设公式，而是故事自己长出来
- **代码实现**: ❌ **写死逻辑** - 4条固定叙事文本随机选择
```typescript
const narratives = [
  `> 第${state.turn}回合。你站在村子里...`,
  `> 夜色更深了。你感到一种莫名的恐惧...`,
  // ... 固定4条
];
const narrative = narratives[Math.floor(Math.random() * narratives.length)];
```
- **问题**: 没有AI生成，没有共振机制，没有记忆影响

## 2. 物理引擎 - 实现状态

### 2.1 引力公式
- **设计**: `F = G × M₁ × M₂ / r² × P × Ω`
- **实现**: ⚠️ **部分实现** - GravityEngine 有完整公式
- **问题**: 但 games.ts 路由中**写死**了简单移动逻辑

### 2.2 质量模型 M = B + S + K + O
| 分量 | 设计 | 实现 | 状态 |
|------|------|------|------|
| B (基础) | 角色预设 | 有 | ✅ |
| S (叙事) | 经历累积 | ❌ **写死为0** | `storyMass: 0` |
| K (关系) | 交互次数 | ⚠️ 有K值但不影响质量 | 未实现 |
| O (道具) | 持有道具 | ❌ **未实现** | - |

### 2.3 环境压强 P = P_history × P_traits
- **设计**: 历史压强 × 特质扰动
- **实现**: ❌ **写死逻辑** - 每回合固定+0.8
```typescript
state.pressure += 0.8;  // 固定增长，无特质影响
```

### 2.4 物理叙事现象
| 现象 | 设计 | 实现 | 状态 |
|------|------|------|------|
| 引力捕获 | 路人S/K飙升变生死之交 | ❌ 未实现 | - |
| 质量坍缩 | NPC死亡M变负 | ❌ 未实现 | - |
| 双星系统 | 两高M NPC互吸 | ❌ 未实现 | - |
| 全局坍缩 | Ω指数增长 | ⚠️ 部分实现 | 有指数增长 |

## 6. 写死逻辑汇总

### 6.1 数值写死
```typescript
// games.ts
state.pressure += 0.8;                    // 固定压强增长
state.omega += 0.02;                      // 固定Ω增长
state.player.states.hunger += 5;          // 固定饥饿增长
state.player.states.fear -= 10;           // 固定恐惧减少
```

### 6.2 文本写死
```typescript
// 4条固定叙事
const narratives = [...];  // 固定4条

// 固定结果文本
result = '你在村子里走了一圈，发现了一些有趣的东西。';
result = '你休息了一会儿，感觉稍微平静了一些。';
```

### 6.3 选择写死
```typescript
// 固定3个基础选择
const choices = [
  { id: 'explore', text: '探索周围环境' },
  { id: 'rest', text: '找个地方休息' },
  { id: 'observe', text: '观察附近的人' }
];
```

### 6.4 天气写死
```typescript
// 仅根据小时判断，无随机性
if (hour >= 6 && hour < 18) {
  state.weather = 'clear';
} else if (hour >= 20 || hour < 5) {
  state.weather = 'night';
} else {
  state.weather = 'fog';
}
```

## 8. 总结

| 类别 | 数量 | 说明 |
|------|------|------|
| 完全实现 | 2 | 基础数据结构、HTTP API |
| 部分实现 | 4 | 物理引擎(有代码未使用)、质量模型、回合推进、存档 |
| 写死逻辑 | 12+ | 叙事、选择、数值、天气、结果等 |
| 未实现 | 10+ | AI生成、道具、精英NPC、事件系统、氛围等 |

**核心问题**: 重构后的代码是一个**可运行的框架**，但**游戏逻辑大量写死**，距离设计文档中的"涌现式叙事"还有很大差距。
