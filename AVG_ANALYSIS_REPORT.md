# 72Hours 项目问题分析报告

## 参考的AVG游戏设计资料

### 1. 交互式叙事游戏核心原则

**来自 Narrative Design 102: Interactive Story Techniques**
- **有意义的选择**：每个选择都应该有意义，而不是虚假分支
- **玩家代理感**：玩家应该感觉自己的决定影响故事
- **反馈循环**：选择后要有明确的反馈（状态变化、叙事变化）

**来自 How to Create Branching Narratives in Interactive Fiction**
- **黄金路径**：需要一个清晰的主线，分支最终要汇聚
- **状态追踪**：使用变量追踪玩家选择，影响后续叙事
- **避免组合爆炸**：不要为每个选择都创建独立分支

### 2. AVG游戏技术架构参考

**来自 AVG.js 和 IntelliVNG Studio**
- **脚本系统**：基于事件的脚本驱动叙事
- **状态管理**：清晰的游戏状态管理（场景、角色、变量）
- **资源管理**：图片、音频、文本资源的统一管理
- **存档系统**：支持多档位存档/读档

**来自 橙光游戏AVG框架**
- **UI系统**：对话框、选项框、角色立绘的管理
- **剧情树**：可视化的剧情分支编辑
- **多线剧情**：主线+支线的结构管理

## 72Hours 当前问题分析

### 🔴 严重问题

#### 1. **缺乏有意义的选择反馈**

**问题描述**：
- 玩家选择后，只有简单的结果描述
- 没有显示状态变化（恐惧、饥饿、攻击性等）
- 选择对后续叙事的影响不透明

**参考标准**：
- 应该显示：「恐惧+10」「与母亲的关系-5」等明确反馈
- 选择应该影响后续可用的选项

**建议改进**：
```javascript
// 选择结果显示
{
  text: "你安慰了母亲",
  stateChanges: {
    fear: -5,
    motherKnot: +3
  },
  unlocks: ["ask_mother_secret"],  // 解锁新选项
  blocks: ["accuse_mother"]         // 封锁某些选项
}
```

#### 2. **没有存档/读档系统**

**问题描述**：
- 游戏必须一次玩完72回合
- 刷新页面就丢失进度
- 无法回溯关键选择

**参考标准**：
- 橙光游戏：随时存档，多档位
- 传统AVG：章节自动存档

**建议改进**：
- 每回合自动保存到 localStorage
- 提供「回到上一回合」功能
- 关键节点手动存档

#### 3. **叙事与游戏机制脱节**

**问题描述**：
- 叙事文本（narrative）和系统状态（压强、Ω）关联弱
- 玩家看不懂「压强82」意味着什么
- Ω值变化没有叙事体现

**参考标准**：
- 状态应该通过叙事体现，而不是只显示数字
- 例如：压强高时，叙事应该更紧张、选项更极端

**建议改进**：
```javascript
// 根据压强调整叙事风格
if (pressure > 70) {
  narrativeStyle = "紧张、急促、充满威胁";
  choices = ["逃跑", "战斗", "隐藏"];  // 极端选项
} else {
  narrativeStyle = "平静、日常";
  choices = ["交谈", "观察", "等待"];  // 温和选项
}
```

### 🟡 中等问题

#### 4. **缺乏视觉反馈**

**问题描述**：
- 纯文本界面，没有视觉变化
- 没有角色立绘、场景背景
- 天气、时间变化没有视觉体现

**参考标准**：
- 即使是文字AVG，也有背景图、角色立绘
- 天气/时间变化通过背景和色调体现

**建议改进**：
- 添加简单的场景背景（CSS渐变或图片）
- 根据天气/时间改变色调
- NPC出现时显示简单的头像或图标

#### 5. **选择缺乏策略性**

**问题描述**：
- 3个选择通常只有1个是"正确"的
- 没有隐藏选项的触发条件
- 选择后果不可预测

**参考标准**：
- 应该有多种 valid 策略（激进/保守/平衡）
- 隐藏选项需要特定条件触发
- 选择后果应该有一定可预测性

**建议改进**：
```javascript
// 隐藏选项示例
{
  id: "bribe_officer",
  text: "贿赂官兵",
  condition: {
    hasItem: "silver",
    trait: "worldly"
  },
  hidden: true  // 不满足条件时不显示
}
```

#### 6. **NPC互动深度不足**

**问题描述**：
- NPC只有名字，没有立绘、性格展示
- 羁绊值（K值）变化不可见
- 没有NPC专属剧情线

**参考标准**：
- 每个NPC应该有独立的好感度/羁绊系统
- NPC有自己的故事线，可以解锁
- NPC状态会随游戏进程变化

**建议改进**：
- 添加NPC面板，显示羁绊值、当前状态
- NPC有自己的执念和故事
- 高羁绊时解锁NPC专属剧情

---

### 🟢 轻微问题

#### 7. **缺少游戏引导**

**问题描述**：
- 新玩家不理解「压强」「Ω」是什么意思
- 不知道游戏目标是什么
- 没有教程或说明

**建议改进**：
- 添加「游戏说明」按钮
- 第一次游戏时显示简单教程
- 鼠标悬停显示术语解释

#### 8. **音效和氛围缺失**

**问题描述**：
- 完全没有音效
- 没有背景音乐
- 氛围全靠文字

**建议改进**：
- 添加简单的背景音乐（根据场景变化）
- 选择时的音效反馈
- 紧张场景的心跳声等氛围音效

#### 9. **移动端适配不佳**

**问题描述**：
- 界面是为桌面设计的
- 手机上按钮太小
- 没有触摸优化

**建议改进**：
- 添加响应式设计
- 增大移动端按钮尺寸
- 支持滑动手势

## 优先级建议

### P0（必须修复）
1. 添加存档/读档系统
2. 增强选择反馈（显示状态变化）
3. 叙事与状态系统关联

### P1（应该修复）
4. 添加视觉反馈（背景、色调变化）
5. 增加选择策略性（隐藏选项、条件）
6. 深化NPC系统

### P2（可以优化）
7. 添加游戏引导
8. 添加音效
9. 移动端适配
